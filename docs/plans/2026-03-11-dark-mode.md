# Dark Mode Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:executing-plans to implement this plan task-by-task.

**Goal:** Add per-guide light/dark theme toggling for guides that have a clear dark variant, plus dark mode for the sidebar/layout chrome.

**Architecture:** Add a `DarkCSSVars` field to the `Guide` struct. When non-nil, the guide supports dark mode. Each guide's `<style>` block emits dark vars under `[data-theme="dark"]`. Alpine.js manages a `data-theme` attribute on `<html>`, persisted to `localStorage`. The sidebar and layout use CSS custom properties so they also respond to the toggle. A toggle button in the sidebar lets users switch modes.

**Tech Stack:** Go, Templ, Alpine.js (toggle state + localStorage), Tailwind CSS v4, CSS custom properties

**Guides getting dark mode:** Brutalist, Minimal, Cassette, Bento, Swiss, Newspaper, Tracker (7 of 10)
**Guides skipped:** Terminal (already dark), Retro OS (specific aesthetic), Glass (already dark)

### Task 1: Add DarkCSSVars to Guide struct and BuildDarkCSS helper

**Files:**
- Modify: `guides/registry.go`

**Step 1: Add DarkCSSVars field to Guide struct**

Add after the `CSSVars` field:

```go
// DarkCSSVars holds dark-mode overrides for CSS custom properties.
// When non-nil, the guide supports a dark mode toggle.
// Only color/shadow/border tokens that change need to be specified —
// typography, layout, and radius tokens carry over from CSSVars.
DarkCSSVars map[string]string
```

**Step 2: Add BuildDarkCSS helper function**

Add after `BuildCSSVars`:

```go
// BuildDarkCSS generates a [data-theme="dark"] block for guides with dark variants.
// Returns an empty string if darkVars is nil.
func BuildDarkCSS(darkVars map[string]string) string {
	if darkVars == nil {
		return ""
	}
	return "[data-theme=\"dark\"]{" + BuildCSSVars(darkVars) + "}"
}
```

**Step 3: Add HasDarkMode convenience method**

```go
// HasDarkMode reports whether this guide supports dark mode toggling.
func (g Guide) HasDarkMode() bool {
	return g.DarkCSSVars != nil
}
```

### Task 4: Add dark mode toggle to sidebar

**Files:**
- Modify: `templates/sidebar.templ`

**Step 1: Add a dark mode toggle button in the sidebar footer**

In `sidebar.templ`, add a toggle button in the footer `<div>` (before the GitHub link). The toggle uses Alpine.js to:
- Read initial state from `localStorage`
- Toggle `data-theme` on `document.documentElement`
- Persist to `localStorage`
- Only show when the current guide has dark mode (pass via a new attribute or always show)

Since the sidebar doesn't know which guide is active at render time (it uses Alpine `active` state), and dark mode is a global preference, the toggle should always be visible. Guides without dark mode simply won't have dark CSS vars, so the toggle has no effect on them — that's acceptable and keeps it simple.

Add this Alpine component in the sidebar footer div, before the GitHub link `<a>`:

```templ
<div
    x-data="{ dark: localStorage.getItem('theme') === 'dark' }"
    x-init="if (dark) document.documentElement.setAttribute('data-theme', 'dark')"
    class="flex items-center justify-between"
>
    <span class="text-xs text-gray-400">Theme</span>
    <button
        class="relative w-10 h-5 rounded-full transition-colors duration-200"
        { templ.Attributes{":class": "dark ? 'bg-gray-600' : 'bg-gray-300'"}... }
        { templ.Attributes{"@click": "dark = !dark; if (dark) { document.documentElement.setAttribute('data-theme', 'dark'); localStorage.setItem('theme', 'dark') } else { document.documentElement.removeAttribute('data-theme'); localStorage.setItem('theme', 'light') }"}... }
        aria-label="Toggle dark mode"
    >
        <span
            class="absolute top-0.5 left-0.5 w-4 h-4 bg-white rounded-full shadow transition-transform duration-200"
            { templ.Attributes{":class": "dark && 'translate-x-5'"}... }
        ></span>
    </button>
</div>
```

**Step 1: Add inline script to prevent flash of wrong theme**

Add a `<script>` tag in the `<head>` (after the Alpine script, before `</head>`) that reads localStorage and sets `data-theme` before the page renders. This prevents a flash of light theme when the user has dark mode saved.

```templ
<!-- Prevent flash of wrong theme on page load -->
<script>
    if (localStorage.getItem('theme') === 'dark') {
        document.documentElement.setAttribute('data-theme', 'dark');
    }
</script>
```
